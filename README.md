Що це таке
SmartOrder — це two-sided інструмент: термінал для персоналу і Telegram-інтерфейс для менеджера. Разом вони замінюють WhatsApp-групи, паперові записи і ручне зведення замовлень.
Термінал персоналу (web, задеплоєний на Vercel)
Два окремих підрозділи — Бар і Ресторан — мають свої термінали. Кожен працівник відкриває потрібну сторінку і:

Вводить замовлення у вільній формі — без форматування, просто текстом. Наприклад: "2 пляшки джину Hendricks, 5кг яблук, кока-кола 6 шт"
Відправляє — AI автоматично розпізнає товари, визначає постачальника і зберігає кожну позицію окремо
Бачить підтвердження — список розпарсених товарів з можливістю уточнити

Якщо товар вже є в активному замовленні — система це показує і пропонує два варіанти: скасувати дублікат або додати кількість до наявного.
Telegram-бот (інтерфейс менеджера / власника)
Менеджер відкриває бота і отримує повний контроль над замовленнями:

Будь-яке повідомлення або /digest → бот показує зведений дайджест: всі pending-позиції по постачальниках, з автоматичним мержем бару і ресторану. Якщо один товар замовляли обидва підрозділи — це позначається окремо (🔀)
Натиснути на постачальника → список усіх товарів від цього постачальника із зазначенням кількості і звідки прийшло замовлення
«📤 Відправити постачальнику» → повідомлення автоматично надсилається постачальнику. У продакшн-версії — на особистий Telegram або інший месенджер постачальника. У демо-версії — в публічний Telegram-канал для суддів. Всі товари переходять у статус sent, формується нове чисте замовлення
«🔓 Залишити відкритим» → товари залишаються в статусі pending, персонал може дозамовити. При наступному дайджесті нові позиції зіллються з поточними
«📤 Відправити ВСІМ» → один клік відправляє замовлення всім чотирьом постачальникам одночасно

[bar.html / restaurant.html]
        │
        │  POST { venue: "bar"|"restaurant", items: "сирий текст" }

[SmartOrder_WF-A]  ←── Webhook: /webhook/smartorder-demo
        │
        ├─ Read Products Master (Supabase)
        ├─ AI Parse (Claude Haiku) — матчинг товарів + typo handling
        ├─ Map to Rows (Code node)
        └─ Insert → orders_parsed (status: pending)
                        │
                        │
[SmartOrder_WF-B]  ←── Telegram Trigger (менеджер пише боту)
        │
        ├─ /digest або текст  →  дайджест по постачальниках + кнопки
        ├─ view_*             →  деталі постачальника
        ├─ send_*             →  надіслати в канал → status: sent
        ├─ send_all           →  надіслати всім одразу
        └─ keep_open          →  залишити pending (для дозамовлення)
